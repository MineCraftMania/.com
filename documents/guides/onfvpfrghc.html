Gargantuan Christmas Pack

The remastered Christmas resource pack!(⊞)This is a resource pack that is perfect for Christmas and Winter Time as it makes the entire world snowy, adds christmas presents, skiing villagers, and more!

Setting up and More Pictures

How to Install This Resource Pack:

You will have a .zip file for this Resource Pack Then You will have to Open the Resource Packs folder. Open the folder in which you have downloaded this Resource Pack File.Copy the downloaded .zip file. After it then open your Minecraft. Then you will have to Click on the Options. After it then you will have to run Minecraft after placing the .zip file. )

{
  "pack": {
    "pack_format": 0,
    "description": "description"
  }
}

Once you pasted it, you need to replace 0 with the version code you're making you pack for, the codes are:

  • 1 for versions 1.6.1 - 1.8.9
  • 2 for versions 1.9 - 1.10.2
  • 3 for versions 1.11 - 1.12.2
  • 4 for versions 1.13 - 1.14.4
  • 5 for versions 1.15 - 1.16.1
  • 6 for versions 1.16.2 - 1.16.5
  • 7 for version 1.17
  • 8 for version 1.18

After you've changed that, you can edit the description by replacing description with whatever you desire, note that is possible to use color and symbols, to use symbols you use \u####, replace #### with the hex code of the symbol, here's a list of symbols you can use. When you do this, save it as pack.mcmeta, make sure the extension is .mcmeta and not .txt

This step is optional, but Minecraft also allows for you to add a thumbnail for your pack, it must be a picture named pack.png and works with any 1:1 resolution, although a dimension of 128x128 px will display pixel perfect.

With this done, the structure of your folder should look like this:

Each one of these folders modifies the following graphic aspects of the game:

  • blockstates: block models use a JSON file to determine variants to use under certain conditions.
  • font: contains a configuration file to specify some font behaviors.
  • lang: contains the language translation files.
  • models
  • block: contains the block models that render into the world.
  • item: contains the item models that render in your hand, item frames, etc.
  • particles: contains the configuration files that control particle frames.
  • texts: contains the credits, end, and main title splash texts.
  • sounds*: this folder is not generated by default, should contain all the custom sounds.
  • textures
  • block: contains the block textures.
  • colormap: contains the colormap textures used to tint grass, foliage, etc.
  • effect* (unused)
  • entity: contains the entity and tile entity textures.
  • environment: contains environment textures such as the sun, moon, weather, etc.
  • font: contains all the font maps and glyphs used.
  • item: contains the item textures.
  • map: contains the map, waypoints, and markers textures.
  • misc: contains miscellaneous textures such as the pumpkin overlay or spyglass scope.
  • mob_effect: contains the status effect icons (speed, health boost, etc).
  • models: contains the armor textures.
  • painting: contains the painting textures.
  • particle: contains all the particle texture files.

Great! Now you only have to make a new folder in .minecraft/resourcepacks and name it however you want your pack to be named, then proceed to create a pack.mcmeta file like before. At this point, you can copy all the elements you want to modify from the template folder to your pack folder, for example; if I want to modify the bookshelf model, I'll go to the template pack, then look for the models folder and copy bookshelf.json into my new pack keeping the folder structure the same.
